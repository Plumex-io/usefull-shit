2 Auto Layout Критично

Проблема: Багато секцій НЕ використовують Auto Layout. Елементи розставлені вручну з абсолютними x/y координатами.

Приклад: що вже добре

Ці фрейми вже мають Auto Layout — Claude їх чудово конвертує:

FAQ Accordion: Frame 2121453676 — VERTICAL, gap: 0, 8 children (FAQ items)
Cash In Out Cards: Frame 2121453667 — VERTICAL, gap: 10, padding: 40 0
Travel Cards: cards — HORIZONTAL, gap: 24, padding: 52 0 16 40

Як конвертувати фрейм на Auto Layout

  1. Виділи фрейм
  2. В правій панелі натисни + біля "Auto layout" (або Shift+A)
  3. Обери напрямок (Vertical / Horizontal)
  4. Встанови gap (відстань між елементами)
  5. Встанови padding
  6. Для дітей: обери Fill (розтягнути) або Hug (по контенту) або Fixed

3 Структура сторінки Високий

Проблема: Блоки лежать в SECTION "Final" без чіткої ієрархії. Назви на кшталт iPad Pro, 3, Picture. Немає єдиного кореневого фрейму зі всіма секціями.

Рекомендована структура

Page: Landing final
Desktop (1440px) — FRAME, Auto Layout VERTICAL, W: 1440, gap: 0
  ├── Hero                    — FRAME, AL VERTICAL, FILL width
  ├── Parallax Phone          — FRAME, AL VERTICAL, FILL width
  ├── Earn Spend CTA          — FRAME, AL VERTICAL, FILL width
  ├── Travel Cards Carousel   — FRAME, AL HORIZONTAL, overflow
  ├── Cash Access             — FRAME, AL VERTICAL, FILL width
  ├── Cash In Out             — FRAME, AL VERTICAL, FILL width
  ├── Cryptoheads             — FRAME, AL VERTICAL, FILL width
  ├── Invite Club             — FRAME, AL VERTICAL, FILL width
  ├── Security                — FRAME, AL VERTICAL, FILL width
  ├── FAQ                     — FRAME, AL VERTICAL, FILL width
  └── Footer                  — FRAME, AL VERTICAL, FILL width

Mobile (375px) — FRAME, Auto Layout VERTICAL, W: 375, gap: 0
  ├── Hero                    — ті самі назви секцій
  ├── Parallax Phone
  ├── ...
  └── Footer
Ключові принципи:
  • Один кореневий фрейм на breakpoint (Desktop / Mobile)
  • Кореневий фрейм = Auto Layout VERTICAL, gap: 0
  • Кожна секція = дитина з FILL по ширині
  • Назви секцій однакові в Desktop і Mobile

4 Компоненти Високий

Проблема: Карточки Travel, Cash In Out, Invite Club, секційні заголовки — все це звичайні фрейми, що дублюються. Claude створить окремий код для кожного.
Чому: Коли Claude бачить INSTANCE (екземпляр компонента), він автоматично створює один React компонент і переиспользовує його. Без компонентів — дублює JSX для кожної карточки окремо.

Що вже є (добре)

Buttons — компонент кнопки, використовується в Hero, CTA
list web — компонент FAQ item, 8 інстансів в акордеоні

Що потрібно створити

КомпонентВаріанти (properties)Де використовується
Card Theme: Light / Dark / Image
Size: SM / LG
Has Image: true / false 		Travel Cards (6x), Cash In Out (4x), Invite Club (4x)
Section Header Has Badge: true / false
Has Subtitle: true / false 		Cash Access, Cash In Out, Cryptoheads, Security
Email Form State: Default / Focus / Typing / Error / Success Invite Club
Footer Column — Footer (4 колонки)
Chip / Badge Label: text Parallax (Swap, Send, Cash, Buy), Cash Access badge

Як створити компонент у Figma

  1. Виділи фрейм → Ctrl/Cmd + Alt + K (Create Component)
  2. Для варіантів: виділи кілька компонентів → "Combine as Variants"
  3. Додай Component Properties (текст, boolean, instance swap) в правій панелі
  4. Перемісти компоненти на окрему сторінку Components

5 GROUP → FRAME Високий

Проблема: Є GROUP вузли (наприклад Group 1597881458). Groups не підтримують Auto Layout, padding, constraints, fill/stroke.

GROUP

  • Без Auto Layout
  • Без padding
  • Без fill/stroke
  • Без constraints
  • Claude не може визначити layout

FRAME

  • Auto Layout
  • Padding, gap, spacing
  • Fill, stroke, effects
  • Constraints, min/max
  • Claude точно конвертує
Як виправити: Виділи GROUP → правий клік → "Frame selection" (Ctrl/Cmd + Alt + G). Потім додай Auto Layout.

8 Breakpoints Середній

Проблема: Є iPad Pro (834px) та iPhone (375px), але немає Desktop (1440px) — основного breakpoint для web.
BreakpointШиринаСтатус
Desktop1440pxНЕМАЄ — потрібно додати
Tablet834px (iPad Pro)Є
Mobile375px (iPhone)Є
Рекомендація:
  • Desktop (1440px) — основний, Claude верстає з нього
  • Mobile (375px) — обов'язковий для адаптиву
  • Tablet (834px) — опціонально, можна зробити через CSS

Правила для breakpoints

  • Назви секцій однакові в Desktop і Mobile (Hero, Footer, etc.)
  • Claude зіставляє секції за назвою — Desktop Hero + Mobile Hero = один адаптивний блок
  • Не називати фрейми іменем пристрою (iPad Pro 11") — називати за контентом

9 Фінальний чеклист

  • Перейменувати всі фрейми семантично Критично

    Жодного Frame 2121453... — тільки осмислені назви

  • Увімкнути Auto Layout на всіх секціях Критично

    Кожна секція, карточка, група — Auto Layout

  • Один кореневий фрейм на breakpoint Високий

    Desktop (1440px) & Mobile (375px) — AL VERTICAL

  • Створити компоненти: Card, Section Header, Email Form Високий
  • Замінити GROUP на FRAME Високий
  • Назвати всі зображення зрозуміло Середній
  • Виправити typo у стилях (bols → bold) Середній
  • Додати Desktop (1440px) фрейм Середній
  • Прив'язати всі тексти/кольори до стилів Середній
  • Заповнити або видалити порожню Variable Collection Середній
